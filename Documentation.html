1. Contexte du Projet

1.1 Présentation de Fortuneo Banque

Fortuneo Banque est une banque en ligne française proposant une gamme variée de services financiers et bancaires. Dans un secteur bancaire hautement compétitif, la rétention client est un enjeu majeur pour assurer la pérennité et la croissance de l'entreprise.

Problématique Business

Le churn (taux d'attrition) représente le pourcentage de clients qui quittent la banque sur une période donnée. Un taux de churn élevé impacte directement :

  • Le chiffre d'affaires et la rentabilité
  • Le coût d'acquisition de nouveaux clients (CAC)
  • La réputation de la marque
  • La valeur vie client (Customer Lifetime Value - CLV)

1.3 Contexte Technologique

Avec l'avènement du Machine Learning et de l'Intelligence Artificielle, il est désormais possible de prédire avec précision quels clients sont susceptibles de quitter la banque. Cette capacité prédictive permet de mettre en place des actions préventives ciblées pour retenir les clients à risque.

2. Objectifs et Intérêt de l'Étude

2.1 Objectif Principal

Mission

Développer un système de prédiction du churn client permettant d'identifier les clients susceptibles de quitter la banque afin de mettre en place des stratégies de rétention personnalisées et efficaces.

2.2 Objectifs Spécifiques

Analytiques

  • Identifier les facteurs clés du churn
  • Comprendre les profils clients à risque
  • Analyser les patterns comportementaux
  • Quantifier l'impact de chaque variable

Techniques

  • Construire un modèle prédictif performant (ROC-AUC > 0.85)
  • Optimiser le F1-Score pour équilibrer précision et recall
  • Garantir l'interprétabilité du modèle
  • Assurer la reproductibilité des résultats

Opérationnels

  • Déployer une API de prédiction en production
  • Segmenter les clients par niveau de risque
  • Automatiser les alertes churn
  • Recommander des actions de rétention ciblées

3. Méthodologie

3.1 Approche CRISP-DM

Le projet suit la méthodologie standard :

  1. Compréhension Business - Besoins métier, KPIs
  2. Exploration Données - Analyse, visualisations, tests
  3. Préparation - Nettoyage, features, encodage
  4. Modélisation - 6 algos, optimisation, CV
  5. Évaluation - Performance, erreurs, recommandations
  6. Déploiement - API, Docker, production

3.2 Dataset

Source Kaggle

Dataset: Binary Classification with a Bank Churn

Taille: 165,000+ observations × 11 variables

Déséquilibre: ~20% churn

Variables

  • CreditScore - Score crédit (300-850)
  • Geography - Pays (France, Spain, Germany)
  • Gender - Genre (Male, Female)
  • Age - Âge (18-100)
  • Tenure - Ancienneté (0-10 ans)
  • Balance - Solde compte
  • NumOfProducts - Nombre produits (1-4)
  • HasCrCard - Carte crédit (0/1)
  • IsActiveMember - Membre actif (0/1)
  • EstimatedSalary - Salaire estimé
  • Exited - Variable cible (0=Resté, 1=Parti)

3.3 Feature Engineering
